# JR 乘車 RECORD

JR 일본 철도 **승차 기록 PWA**. 지도에서 노선을 보고, 직접 탄 구간을 색칠해 기록하는 개인용 웹앱입니다.

- **라이브:** https://pos09009.github.io/jr-riding-record/
- 휴대폰에서 브라우저로 접속 → "홈 화면에 추가"로 앱처럼 설치 (오프라인 동작)

## 기능

- 캔버스 기반 커스텀 지도 (외부 지도 API 없이 동작, 팬/줌)
- 구간별 승차 기록 (지도 팝업 토글 · 수동 범위 입력 · GPS 자동 감지)
- 역/노선 이름 검색 → 해당 역으로 지도 확대
- 백업: 기록 JSON 내보내기 / 가져오기
- PWA: 오프라인 캐시, 홈 화면 설치

## 파일 구조

| 파일 | 역할 |
|------|------|
| `index.html` | 화면·스타일·PWA 메타 (**진입점**) |
| `app.js` | 모든 로직 (지도·GPS·검색·기록) |
| `data/kanto.js` | 간토 노선 데이터 (원본 10개 노선) |
| `data/kanto2.js` | 간토 노선 데이터 (2차 추가 14개) |
| `sw.js` | 서비스워커 (오프라인 캐시) |
| `manifest.json` | PWA 매니페스트 |

데이터는 `<script src>`로 불러와 `window.JR_DATA`에 합쳐집니다 (`file://`에서도 동작하도록 `fetch` 대신 이 방식 사용). 노선 = `{id, name, color, loop, stations:[{n, lat, lng}, ...]}`. 구간 키는 `노선ID_인덱스`이며, 순환선(`loop:true`)은 마지막 역→첫 역 구간이 추가됩니다.

야마노테 · 츄오쾌속 · 츄오·소부완행 · 케이힌도호쿠 · 도카이도 · 쇼난신주쿠라인 · 타카사키 · 우츠노미야 · 조반 · 무사시노 · 사이쿄 · 요코스카 · 소부쾌속 · 요코하마 · 네기시 · 케이요(+지선2) · 난부(+지선) · 카와고에 · 오메 · 이츠카이치 · 하치코

전 노선 시점·종점·역수·좌표·표기를 위키 기준으로 검증 완료.

## 역명 표기 원칙

일본어 **원음 우선**, 단 한국어에서 굳어진 관용 표기는 예외.

- 무성음은 격음으로: 高崎→타카사키(다카사키 ❌), 吉祥寺→키치조지
- 한자 음독 금지: 自治医大→지치이다이, 新都心→신토신
- ㅈ·ㅊ 뒤 요음의 y는 생략 (ㅈ·ㅊ이 이미 구개음): 町ちょう→초, じゅ→주 (쵸·쥬 ❌). 단 ㅅ은 유지(しゃ→샤)
- 장음 생략: 国府津こうづ→코즈 (코우즈 ❌)
- 관용 예외: 도쿄(東京) · 도카이도/케이힌**도**호쿠(東) · 츄오(中央, 노선명만) · 시모우사(下総, 옛 지방명)

## 새 노선/지역 추가

1. `data/지역명.js` 생성 → `window.JR_DATA.지역키 = [ {id, name, color, loop, stations:[...]}, ... ]`
2. `index.html`에 `<script src="data/지역명.js"></script>` 한 줄 추가
3. `sw.js`의 `ASSETS`에 파일 경로 추가 + `CACHE` 버전 숫자 올리기

## 업데이트 반영 (PWA 캐시 주의)

데이터/코드 수정 후 GitHub에 올렸는데 폰에서 안 바뀌면 캐시 문제입니다 (GitHub Pages 약 10분 HTTP 캐시 + 옛 서비스워커). 가장 확실한 방법: **홈 화면에서 앱 삭제 → 브라우저로 재접속 → 다시 홈 화면에 추가**. 파일 수정 시 `sw.js`의 `CACHE` 버전을 항상 올리세요.
